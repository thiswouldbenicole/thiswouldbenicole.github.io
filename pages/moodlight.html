MoodLight

User Experience Designer

January 2014 - May 2014

MoodLight is an interactive ambient lighting system developed in Cornell's Interaction Design Lab. As a research assitant, I redesigned the android app to use to control the system for research and personal usage.

The MoodLight System

"MoodLight is a playful system that uses ambient colored light to provide feedback regarding an individualâ€™s current stress levels, encouraging reflection and non-judgmental self-awareness." Users interact with the system by holding "pips," which are small bluetooth-connected devices that measure a user's Galvanic Skin Response (GSR). GSR levels roughly corespond with stress levels, and Phillips Hue Bulbs in the MoodLight system change color in response to changes in a user's GSR levels. More information about the system can be found here.

Interaction Design

The MoodLight android app was developed to control the above ambient lighting system. Below are some screenshots of the old app, which had convoluted menus that made it hard to change settings between experiment trials.

My redesign of the app needed to not only make it easier to navigate through all of the current features, but also account for two very different use-cases of the app. The main use-case of the app is using it to run in-house experiments run by actual researchers. Researchers need full control over every possible feature of the system.

However, the team hopes to start conducting a new study where people with anxiety disorders can bring the lighting system into their homes and use the system for personal use to de-stress. For this use-case, users should be prompted with as few options as possible to prevent them from feeling even more stressed.

Below are some high-fidelity wireframes of selecting a mode, and the menus of the One-Player, Two-Player, and personal use modes.

Visual Design

First Iterations

Below is a first iteration of the app's visual design.

To handle the two very different use-cases, users are prompted to choose between a research mode and personal use mode when they first open the app. The rest of the app's UI is colored red if the researcher mode is being used, or blue if the personal use mode is being used, to make it easy to tell which mode the app is being used in at a glance.

Final Iterations

The personal use mode will be seen mostly by users with anxiety disorders so it was extremely important to make sure this mode of the app has few controls as possible, to prevent any extra stress. It was also important to make sure the app felt friendly and inviting. Copy is slighly different on the personal use mode to make it friendlier and more personal through usage of more coloquiel lanugage and emoticons.

All possible settings for the selected type of trial are available upfront when using the app as a researcher. The copy might not make sense to a random person looking at the app, but the only people who will generally be using this mode of the app already have a deep underdstanding of the capabilities of the lighting system.

The lighting system can either scroll through an entire rainbow, or cycle between two colors, to signify change in stress levels. When the full-spectrum mode is turned off, a color picker drops down to allow users to pick which two specfic colors they want the system to cycle between.
